# 客戶資料管理系統

這是一個可直接使用的財務型 CRM，包含客戶資料、委刊單、發票追蹤、入款紀錄、合約追蹤與互動追蹤。各模組支援新增、編輯與刪除；完成 Google Apps Script 設定後，資料會同步到 Google Sheet。

## 使用方式

1. 打開 `index.html` 即可先使用系統。
2. 建立一份新的 Google Sheet，複製網址中的 Sheet ID。
3. 到 [Google Apps Script](https://script.google.com/) 建立新專案。
4. 將 `google-apps-script/Code.gs` 的內容貼到 Apps Script 編輯器。
5. 點選「部署」→「新增部署作業」→ 類型選「網頁應用程式」。
6. 執行身分選「我」，存取權選「知道連結的任何人」，完成後複製 Web App URL。
7. 回到系統的「設定」，貼上 Web App URL 與 Sheet ID，按「儲存並測試」。
8. 可按「寫入示範資料」建立範例資料與工作表欄位。

## Google Sheet 工作表

系統會自動建立並維護以下工作表：

- `Customers`：客戶名稱、統一編號、聯絡人、Email、電話、付款條件、地址、備註
- `Orders`：BD、銷售類別、產業、代理商、品牌客戶、報價日期、報價金額、資料形式、資料筆數、交付日期、是否已開發票
- `Invoices`：已收款、訂單編號、請款時程、發票開立、發票號碼、公司、統編、未稅/含稅金額、購買商品、收件資訊、狀態、備註、說明
- `Payments`：入款日期、客戶、對應發票、金額、方式、備註
- `Contracts`：類別、公司名稱、窗口、聯繫方式、客戶狀態、合約類別、合約期間、價格、年約價值、請款模式
- `Activities`：互動日期、客戶、類型、內容、下次追蹤日
- `Options`：下拉選單選項，例如付款條件、BD、銷售類別、產業、發票狀態、付款方式、客戶狀態、請款模式

## 注意事項

- Google Sheet 會作為主要資料庫；瀏覽器本機資料只作為離線示範與同步失敗時的暫存。
- 若修改 Apps Script，部署後要重新發佈新版本，Web App URL 才會使用新版程式。
- 存取權若選得太嚴格，前端會無法寫入 Sheet。
- 若新增、編輯或刪除功能有調整，請重新複製 `google-apps-script/Code.gs` 到 Apps Script，並部署「新增版本」。
- 「設定」裡的選單設定一行代表一個選項，儲存後會同步到 Google Sheet 的 `Options` 工作表。
